Smart Horrocrux

分析

题目合约：

Some security researchers have recently found an eighth Horrocrux, it seems that Voldemort has link to a smart contract, can you destroy it?

意思就是要我们摧毁这个合约。

既然要摧毁这个合约，那肯定与自毁脱不了干系。合约中存在一个external的kill函数，其中就有自毁语句，那我们的目的肯定是调用到这个自毁语句了，而他限制了调用者必须是本合约，那么向上看能看到destroyIt函数中就有本合约的call调用，思路清晰，调用destroyIt函数。

参数包括一个string类型和一个uint256类型，第一个限制要求我们的string参数的值转成bytes32后与spellInBytes相等，由于合约中已经给出了spellInBytes，我们稍加计算即可。

第二个限制要求invincible为false,而在合约创建时由于传入的value为2，所以初始的invincible为true。

而要让invincible变成false就需要合约的余额为1，而fallback函数能够很轻松的让我们把合约中的所有余额给拿出来，看似合约没有payable函数，那我们就需要使用自毁向其传入1 wei即可。

两个限制绕开之后，就来到最后call调用的步骤。

函数选择器的计算为将spellInBytes与我们传入的magic进行减法，最后转成bytes4，也很简单，计算即可，分析完成开始攻击。

攻击

攻击合约：

攻击步骤即传入1wei调用init函数，然后调用att函数即可。

记得在调用att函数时将gas设置的高一点，由于合约中存在call调用的关系，容易发生gas不足call调用失败

Trickster

分析

题目合约：

题目描述：

We might have spotted a honeypot… Can you manage to obtain the real jackpot?.
Hacking casino slot machines is considered illegal.

大意是我们需要获得头奖。

描述说的模糊不清，我们就可以去工厂合约中查询通关条件

注意这里是JackpotProxy中的balance函数而不是该合约的balance

目的很简单了，把存入jackpot的钱都取出来。

很快发现合约本身就提供取钱的函数，虽然有关于调用者的限制，但上面就有一个没有任何限制条件的改变jackpotProxy的函数，那思路就很清晰了。

通过两种方式都可以查出jackpot的地址，we3.js和区块链浏览器，我这里选择了web3.js。查出之后改变jackpotProxy然后输入余额的一半直接调用函数即可。

攻击

无攻击合约，直接攻击。

查出地址后，将jackpotProxy改成我们自己，然后调用claimPrize输入50000000000000即可。
